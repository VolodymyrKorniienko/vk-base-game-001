## ⚠️ Что требуется для получения финансирования

### 1. Деплой на Base Mainnet (КРИТИЧНО)

**Текущее состояние:** Контракт скомпилирован, но не развёрнут.

**Действия:**
```bash
# 1. Добавьте приватный ключ в .env
echo "DEPLOYER_PRIVATE_KEY=your_private_key_here" >> .env

# 2. Задеплойте контракт на Base mainnet
npm run deploy:base

# 3. Сохраните адрес контракта
echo "NEXT_PUBLIC_CONTRACT_ADDRESS=0x..." >> .env
```

**После деплоя:**
- ✅ Верифицируйте контракт на [Base Scan](https://basescan.org/)
- ✅ Добавьте контракт в [Talent Protocol](https://talent.app) для трекинга

### 2. Настройка Builder Code (ERC-8021 Attribution)

**Текущее состояние:** `NEXT_PUBLIC_BUILDER_CODE` пуст.

**Действия:**
1. Зарегистрируйтесь на [base.dev](https://base.dev)
2. Перейдите в **Settings → Builder Code**
3. Создайте builder code для вашего проекта
4. Добавьте в `.env`:
   ```
   NEXT_PUBLIC_BUILDER_CODE=your_builder_code
   ```

**Зачем это нужно:**
- Трекинг активности в экосистеме Base
- Автоматическое начисление Builder Score
- Право на участие в Weekly Rewards

### 3. Настройка CDP Paymaster (спонсируемые транзакции)

**Текущее состояние:** `CDP_PAYMASTER_URL` требует настройки.

**Действия:**
1. Создайте проект в [Coinbase Developer Portal](https://portal.cdp.coinbase.com/)
2. Настройте Paymaster для Base mainnet
3. Получите URL и добавьте в `.env`:
   ```
   CDP_PAYMASTER_URL=https://...
   ```

### 4. Трекинг метрик

**Что нужно отслеживать:**
- 📈 Количество уникальных игроков
- 📈 Количество завершённых уровней
- 📈 Количество отминченных NFT
- 📈 Количество транзакций в контракте

**Рекомендуемые инструменты:**
- [The Graph](https://thegraph.com/) — для индексации событий контракта
- [Dune Analytics](https://dune.com/) — для дашбордов
- [Talent Protocol](https://talentprotocol.com/) — для Builder Score
